# Internationalisation (i18n)

Mops can switch text and locators to match the language of your application. You do not need to add any extra code in your tests: once the translator knows where your catalogue lives, every `Element` can replace placeholders with the translated values.

## Translation catalogue

Translations live inside a JSON file. The easiest layout is a flat mapping:

```json
{
  "login": {
        "button": "Log In",
        "error": "Wrong email or password"
    }
}
```

You can also group strings per locale:

```json
{
  "default_locale": "en",
  "locales": {
    "en": {   
      "login": {
        "button": "Log In",
        "error": "Wrong email or password"
      }
    },
    "es": { 
      "login": {
        "button": "...",
        "error": "..."
      }
    }
  }
}
```

Keys use dot notation so you can organise them by page or feature. Values can be plain strings or nested objects.

## Auto-discovery

`I18nTranslator` tries to find the catalogue for you in this order:

1. Environment variables.
2. The `[tool.mops.i18n]` section inside `pyproject.toml`.

If none of these options works, translation falls back to the original text. No error is raised.

### Environment variables

Set these variables before your tests start:

- `MOPS_I18N_PATH` – absolute path to the JSON file.
- `MOPS_I18N_LOCALE` – locale name to use when the catalogue stores more than one language.

### Pyproject settings

Add configuration inside your `pyproject.toml`:

```toml
[tool.mops.i18n]
path = "resources/i18n.json"
locale = "es"
```

`path` is mandatory. `locale` is optional.

### Programmatic overrides

Call `configure_translator` when you need to override discovery at runtime (for example inside `conftest.py` before any `Element` is created):

```python
from mops.utils.i18n_translator import configure_translator

configure_translator(path='resources/i18n.json', locale='es')
```

Passing no arguments resets the override:

```python
configure_translator()
```

The override takes precedence over environment variables and `pyproject.toml` for the rest of the process.

## Manual configuration

You can create a translator yourself when you want full control in a script or a unit test:

```python
from mops.utils.i18n_translator import I18nTranslator

translator = I18nTranslator()
translator.configure(path='tests/data/i18n.json', locale='es')
print(translator.translate('login.button'))
```

`Element` already creates a shared translator and runs the discovery steps when it needs a translation. Call `configure_translator` only when you want to override that behaviour and point to a specific file:

```python
from mops.base.element import Element

Element.configure_translator(path='resources/i18n.json', locale='en')
```

Every element will then resolve placeholders like `{login.button}` in locators or text arguments using your explicit catalogue.

If you need to avoid translation in Element you have to use avoid_translation kwarg

```python
from mops.base.element import Element

Element('{some.element}', avoid_translation=True)
```

## Failure modes

- Missing file: discovery returns an empty catalogue and translations fall back to original values.
- Invalid JSON: the translator raises `I18nConfigurationError`.
- Missing key: the translator returns the key itself (unless you pass an explicit default value).
